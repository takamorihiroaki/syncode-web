2. [x] 実際のリンクが 404 になるかテスト
3. [x] ユーザーから提供された ID との不整合を確認
4. [ ] 修正の方向性をまとめる

## 調査結果
- `https://syncode.jp/articles/` の "KNOWLEDGE BASE" セクションにおいて、大量の記事が**テキストのみ（リンクなし）**で表示されている。
- ユーザーが例示した ID（#620961B1, #BBACC58C 等）を持つ記事がこの「テキストのみ」のリストに含まれている。
- これらの ID をベースにした URL（例: `/articles/620961B1/`）を直接叩くと **404 Not Found** となる。
- slug が設定されている他の記事（例: `/articles/antigravity-performance-fix/`）は正常に表示される。
- 原因として考えられること：
  - Astro のコンテンツコレクションで、これらの記事ファイルに slug が設定されていない、またはファイル名が slug として認識されていない。
  - あるいは、ビルド時にこれらの記事が「下書き」扱いになっているか、何らかのエラーで生成から除外されているが、リスト表示側には名前だけ残っている。
  - または、パスの形式が変更されたが（IDベースからスラッグベースへ）、リスト側が古い ID を参照しており、かつリンク生成に失敗している。
