# トップページ連載特設セクション 設置プラン

今回の全4記事を「1つの物語」としてパッケージ化し、トップページで強力にアピールするためのプランです。

## 1. デザインコンセプト：Documentary Row

単なる記事カードではなく、**「全4回で完結する連載」**であることを示すための専用コンポーネントを導入します。

- **設置場所**: `index.astro` の「まずはこの記事（Featured）」と「新着情報（All Articles）」の間。
- **デザイン要素**: 
  - シリーズタイトル：「【全4回】AIエージェント共創実録：GA4分析からSEO改善まで」
  - 4つの記事を左から右へ、時系列（Vol.1 → Vol.2 → Vol.3）で並べます。
  - 「技術ガイド」は、Vol.1とVol.2の間に「別冊・技術解説」として差し込む形で、ストーリーとノウハウの連動を強調します。

## 2. 実装内容

### 新規コンポーネント制作
- **ファイル名**: `src/components/SeriesSection.astro`
- **役割**: 連載記事をグループ化して表示する、リッチな横並び（またはグリッド）セクション。
- **特徴**: 各記事に「Vol.1」「Technical Guide」などのバッジを付与し、読む順番を明示します。

### index.astro の更新
- `SeriesSection` をインポートし、新着一覧の直前に挿入します。
- 今回の4記事を特定してコンポーネントに渡すロジックを追加します。

## 3. 実施ステップ

1.  `SeriesSection.astro` コンポーネントの作成（レスポンシブ対応）。
2.  `index.astro` への組み込み。
3.  全体の色調やフォントの微調整（プレミアム感の演出）。
4.  ローカルビルドと表示確認。

---

## ご確認事項

- この専用セクションを設けることで、トップページの情報量は少し増えますが、その分「サイトの核となるコンテンツ」を確実に読者に届けることができます。
- デザインはこのブログのミニマルなトーンを継承しつつ、少しだけ「連載らしさ（特別な枠であること）」を強調する予定です。
