# articles_backup 配下の Markdown ファイルのフロントマター修正計画

`src/content/articles_backup` 配下のすべての `.md` ファイルに対して、フロントマターのキー変更、フォーマット統一、および不要なメタデータブロックの削除を行います。

## 現状の課題
調査の結果、バックアップ配下のファイルには以下のような非標準的な構造が含まれていることが判明しました：
- ファイルの先頭に不完全なメタデータがある（`---` で始まっていない場合がある）
- 本文中にも `---` で区切られた `excerpt: "Auto-generated..."` などの不要なメタデータブロックが複数挿入されている
- キー名が `date` や `image` のままのものと、既に `pubDate` になっているものが混在している

## 修正ルール

### 1. キーの変換
- `date` → `pubDate`
- `image` → `heroImage`
- `excerpt` → `description`

### 2. フォーマット統一
- `pubDate`, `updatedDate` を `YYYY-MM-DD` 形式に統一（引用符の有無は問わず、値のみを修正）
- `heroImage` が空または存在しない場合は項目を削除
- `updatedDate` が存在しない場合は追加しない

### 3. 構造のクリーンアップ
- ファイルの先頭に正しいフロントマター（`---` で囲まれたブロック）を 1 つだけ配置
- 本文中に散在する `excerpt`, `tags`, `category`, `draft` などのメタデータ行、およびそれらを囲む `---` セパレーターをすべて削除し、純粋な本文のみを残す

## 変更対象
- `src/content/articles_backup/*.md` 全ファイル（19件）

## 実施手順

### フェーズ 1: 変換スクリプトの作成
Node.js を使用して、以下の処理を行うスクリプトを作成します：
1. ファイルを読み込み、正規表現またはシンプルなパーサーで「正規のメタデータ」を抽出
2. 正規のメタデータに対して変換ルール（キー変更、日付フォーマット、不要項目削除）を適用
3. 本文から不要なメタデータブロックを完全に除去
4. 新しいフロントマターと本文を組み合わせてファイルを上書き保存

### フェーズ 2: スクリプトの実行と検証
1. テストとして 1 ファイルに対して実行し、内容を確認
2. 全ファイルに対して実行
3. 目視および `grep` 等で、不要な `excerpt` 行が残っていないか、日付が統一されているかを確認

## 確認事項
> [!IMPORTANT]
> 本文中に挿入されている `---` セパレーターとメタデータ（例: `excerpt: "Auto-generated excerpt for build fix."`）は、以前の処理で誤って挿入されたゴミデータと判断し、これらを一括削除して「純粋な本文 + 正しいフロントマター 1 つ」という標準的な形式に修正してもよろしいでしょうか？

## 検証計画
### 自動検証
- `grep "date:"` や `grep "image:"` で古いキーが残っていないか確認
- `grep "---"` で、各ファイルにフロントマター以外のセパレーターが残っていないか確認

### 手動検証
- 変換後のファイルを数件表示し、意図通りのフォーマット（pubDate: YYYY-MM-DD 等）になっているか確認
