# 【ミッション】サイト全域の CTA（導線）統一・最適化プラン

全43記事に配置されている（あるいは配置されるべき）ボタン群を、専用コンポーネント `CtaInline` へと統一し、メディアとしての回遊性とブランド価値を最大化します。

## 1. 現状の分析と改善方針

### 重大な不整合
- **旧スタイルの残存**: `post-1`〜`6`、`chapter-1`〜`4` 等に、`cta-box` という手書きの `div` 構造が残っています。
- **導線の欠如**: 刷新した「技術検証」系記事の多くに、読了後のアクションを促すボタンが配置されていません。
- **配置のノイズ**: 短い記事に2つのボタンがあるなど、知的な緊張感を削ぐ配置が見られます。

### 改善プロトコル
- **コンポーネントの統一**: すべて `CtaInline`（Recommendation 付き）に置換。
- **配置数の方針**:
    - **2,000文字未満の記事**: 記事末尾に **1つ**（一点突破）。
    - **2,000文字以上の長文記事**: 中盤（問題提起の直後）と末尾に **計2つ**。
- **リンク先の精密設定**: 読者が次に必要とする「実践知」へのリンクを、記事ごとに再定義。

## 2. 実装フェーズ

### フェーズ A：既存ボタンの置換（16記事）
`post-1`〜`6`、`chapter` シリーズ、`markdown-guide` 等に残る `cta-box` を、`CtaInline` に書き換え。
- **[MODIFY]** `src/content/articles/post-1.mdx` ... 他15ファイル

### フェーズ B：新規導線の追加（約25記事）
解説記事、AI戦略系など、現在ボタンがない記事に、文脈に沿った `CtaInline` を新規挿入。
- **[MODIFY]** `src/content/articles/antigravity-start.mdx` 等

### フェーズ C：インポートの整合性確認
全 MDX ファイルで `import CtaInline from '../../components/CtaInline.astro';` が正しく記述されているか点検。

## 3. 検証プラン
- **ビルドテスト**: `npm run build` による構文エラー（インポート漏れ等）のチェック。
- **表示確認**: 代表的な記事において、モバイル/デスクトップ双方で `CtaInline` が美しく表示されるか目視確認。
