# セキュリティ強化：SSH鍵認証への移行とパスワードの削除

プロジェクトからパスワード情報を排除し、より安全でスムーズなデプロイを実現するため、SSH鍵認証（公開鍵認証）への移行を実施します。

## ユーザー確認事項

### 作業の安全性について
> [!IMPORTANT]
> この作業では新しいSSH鍵ペアを生成します。既存の他のサーバー接続に影響を与えることはありません。また、既存のパスワードベースのスクリプトは、新方式での動作確認が完了するまで保持し、最終的に安全に削除します。

### 作業の流れ
1.  **SSH鍵の生成**: ローカル環境で「秘密鍵」と「公開鍵」のペアを作成します。
2.  **公開鍵のサーバー登録**: 提供いただいたパスワードを使用して、一度だけ自動でサーバーへ公開鍵を書き込みます。
3.  **動作確認**: パスワードなしでサーバーに接続できるかテストします。
4.  **スクリプトの刷新**: `npm run deploy` がパスワードなしの `rsync` を使用するように更新します。
5.  **情報のクリーンアップ**: `upload_to_sakura.expect` を削除し、パスワードを完全に消去します。

## 提案する実装内容

### [1] SSH鍵の生成と登録

- `ssh-keygen` を使用して、強度が高く互換性のある `ed25519` 形式の鍵を生成します。
- 一時的なスクリプトを用いて、`syncode.sakura.ne.jp` の `~/.ssh/authorized_keys` に公開鍵を追加します。

### [2] デプロイスクリプトの更新

#### [MODIFY] `scripts/deploy.sh`
- `expect` スクリプトの呼び出しを、直接 `rsync` を使用する形式に変更します。
- 例: `rsync -avz --delete ./dist/ syncode@syncode.sakura.ne.jp:www/`

#### [DELETE] `scripts/upload_to_sakura.expect`
- パスワードが含まれているこのファイルは、移行完了後に削除します。

## 修正後の実行フロー

1.  **鍵の生成 & 送信**: 私が鍵を作成し、サーバーへ登録します。
2.  **接続テスト**: パスワードを求められずに接続できることを確認します。
3.  **スクリプト更新**: デプロイコマンドを刷新します。
4.  **デプロイ実行**: `npm run deploy` で動作を確認します。
5.  **削除**: 旧スクリプトとパスワード情報を消去します。

## 検証プラン

### 1. 認証テスト
- `ssh -o BatchMode=yes syncode@syncode.sakura.ne.jp exit` を実行し、パスワード入力なしで正常終了するか確認します。

### 2. デプロイテスト
- 実際にファイルを一つ更新し、`npm run deploy` でサーバーに反映されるか確認します。
