# 全会話ログ（.mdアーティファクト）の一括エクスポート計画

「Select a conversation」に表示されているすべての過去の会話に含まれるドキュメント（.mdファイル）を一括で収集し、ユーザーがアクセスしやすい形でワークスペースに出力します。

## 概要
Antigravityの内部ディレクトリに分散して保存されている各セッションのアーティファクト（Implementation Plan, Walkthrough, Task）を、IDごとに整理して一括抽出します。

## 実施手順

### 1. 会話タイトルの特定とマッピング
各フォルダ（会話）に含まれる `implementation_plan.md` や `walkthrough.md` の冒頭を確認し、「Select a conversation」メニューに表示されているタイトルを特定します。
また、現在の会話履歴（直近10件）のデータも併用して、IDとタイトルの対応表を作成します。

### 2. データの収集と整理
特定したタイトルに基づき、以下のディレクトリから最新の `.md` アーティファクトを抽出します。
- 対象項目: Implementation Plan, Walkthrough, Task, Scratchpads

### 2. 出力先の作成
ワークスペースのルートに、エクスポート用のディレクトリを作成します。
- 出力パス: `/Volumes/TOSHIBA EXT/syncode-web/conversation_archives/`

### 4. ファイル名の命名規則
各フォルダに「表示タイトル」を使用し、その中に各ドキュメントを保存します。
- `conversation_archives/[表示タイトル]_[ID]/[種類].md`
例: `conversation_archives/Deleting_Image_From_Markdown_6da351fc/Walkthrough.md`

## ユーザーへの確認事項
> [!IMPORTANT]
> - 過去の履歴が膨大（約45セッション）なため、ファイル数が多くなりますが、すべて一括でエクスポートしてよろしいでしょうか？
> - ファイル名に「会話のタイトル」を入れたいのですが、メタデータから取得できない場合はID（UUID）がフォルダ名になります。あらかじめご了承ください。

## 検証計画
- 指定した出力ディレクトリに、すべての会話IDに対応するフォルダとファイルが生成されているか確認します。
- ファイルの中身が正しくコピーされているか、数件をピックアップして検証します。
