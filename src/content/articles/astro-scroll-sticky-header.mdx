## 読者を迷わせない、美しき「追従」の実装。

コンテンツが縦に長くなるほど、読者のナビゲーション体験は希薄になります。スクロールに合わせて姿を変え、常に「次への導線」を差し出す固定ヘッダーは、プロフェッショナルなメディアの必須条件です。

AstroのバニラJSとCSSだけで、ロゴの移動から背景の透過までを滑らかに制御する。UXを飛躍させ、サイトの回遊率を高めるための「Sticky Header」実装術を徹底解説します。
Astroで「スクロールに連動して姿が変わる固定ヘッダー」は、<strong>外部ライブラリ不要・バニラJSのみ</strong>で実装できます。

実装時間は約30分。ヘッダー関連ファイルは1つだけ（`BaseLayout.astro`）触れば完成します。
---

## なぜ実装しようと思ったか

サイトに記事が10本を超えてきたとき、あることに気づきました。

「読者が記事を読みながら、別のページへ移動する手段がない」

ナビゲーションはページ最上部にあるため、記事を読み進めていくとスクロールの彼方に消えてしまいます。  
読み終わった読者が「他も読もうかな」と思っても、戻る手段がない状態です。

これは回遊率の低下に直結します。  
<strong>記事が増えてきた今こそ、ナビを"常に見える場所"に置く必要がある</strong>と判断しました。

---

## 設計の考え方：ナビ項目は増やさない

実装前に一つ考えたのが、「カテゴリ絞り込みのリンクもヘッダーに追加すべきか？」という点です。

結論は「追加しない」でした。理由は3点。

- TOPページにすでにカテゴリフィルター（All / AIツール / 初期設定...）が存在する
- ヘッダーに8〜9項目並ぶと「どれが何のためのナビ？」と読者が混乱する
- SYNCODEのミニマルな世界観を壊したくない

```
スクロール前：  SYNCODE（中央大）
               TOP  ABOUT  WORK  TOOLS（下段）

スクロール後：  SYNCODE   TOP ABOUT WORK TOOLS（横一列・固定）
```

シンプルに保つことが、最終的には読者の行動を促します。

## 実装手順

### 1. ヘッダーのHTML構造を確認する

`BaseLayout.astro` のヘッダー部分はこの形になっています。

```html
<header class="site-header">
    <div class="header-inner">
        <h1 class="logo"><a href="/">SYNCODE</a></h1>
        <nav class="main-nav">
            <ul>
                <li><a href="/">TOP</a></li>
                <li><a href="/about">ABOUT</a></li>
                <li><a href="/works">WORK</a></li>
                <li><a href="/tools">TOOLS</a></li>
            </ul>
        </nav>
    </div>
</header>

<!-- レイアウトずれ防止のスペーサー -->
<div id="header-spacer" style="height: 0;"></div>
```

スペーサー `div` がポイントです。ヘッダーが `position: fixed` になった瞬間、その高さ分だけコンテンツが上にズレます。このスペーサーがその分の余白を補います。

---

### 2. JavaScriptでスクロール量を監視する

スクロール量が80pxを超えたら、ヘッダーに `is-scrolled` クラスを付与します。  
外れたら外す。それだけです。

```js
const header = document.querySelector('.site-header');
const spacer = document.getElementById('header-spacer');

const SCROLL_THRESHOLD = 80;

function updateHeader() {
    if (!header || !spacer) return;

    if (window.scrollY > SCROLL_THRESHOLD) {
        header.classList.add('is-scrolled');
        spacer.style.height = header.offsetHeight + 'px';
    } else {
        header.classList.remove('is-scrolled');
        spacer.style.height = '0px';
    }
}

window.addEventListener('scroll', updateHeader, { passive: true });
updateHeader(); // ページロード時にも即時適用
```

`{ passive: true }` はスクロールのパフォーマンス最適化オプションです。  
`updateHeader()` をリスナー登録後に即呼び出しているのは、ページをスクロール済みの状態でリロードしたときに正しい初期状態を適用するためです。

---

### 3. CSSで2つの状態を定義する

`.site-header`（通常時）と `.site-header.is-scrolled`（固定時）の2セットを書きます。

```css
/* 通常状態：中央寄せ・縦並び */
.site-header {
    padding: 50px 0 40px;
    text-align: center;
    background: #fff;
    transition: box-shadow 0.3s ease;
}

.site-header .header-inner {
    display: flex;
    flex-direction: column;
    align-items: center;
}

.site-header .logo {
    font-size: 1.6rem;
    margin-bottom: 24px;
    transition: font-size 0.3s ease, margin 0.3s ease;
}

/* スクロール後：固定・横並び・すりガラス背景 */
.site-header.is-scrolled {
    position: fixed;
    top: 0;
    left: 0;
    right: 0;
    padding: 0;
    background: rgba(255, 255, 255, 0.96);
    backdrop-filter: blur(10px);
    border-bottom: 1px solid #f0f0f0;
    box-shadow: 0 2px 20px rgba(0, 0, 0, 0.06);
}

.site-header.is-scrolled .header-inner {
    flex-direction: row;
    justify-content: space-between;
    align-items: center;
    height: 60px;
    max-width: 1100px;
    margin: 0 auto;
    padding: 0 8%;
}

.site-header.is-scrolled .logo {
    font-size: 1.1rem;
    margin-bottom: 0;
}
```

`backdrop-filter: blur(10px)` がすりガラス効果の正体です。  
`rgba(255, 255, 255, 0.96)` と組み合わせることで、下のコンテンツがうっすら透けて見えるモダンな印象になります。

## よくあるミスと対処

ページがガタつく場合は、スペーサーの実装を確認してください。ヘッダーが `fixed` になった瞬間、その分のスペースが空白になるため、スペーサーで高さを補う必要があります。

```js
// ヘッダーの実際の高さをスペーサーに反映する
spacer.style.height = header.offsetHeight + 'px';
```

固定後のヘッダー高さをリアルタイムに取得して渡しているため、フォントサイズや padding が変わっても自動的に追従します。

## まとめ

実装のポイントは3つです。

1. ヘッダーに `is-scrolled` クラスを付け外しするだけ
2. `position: fixed` 時のスペーサーを忘れない
3. CSS遷移は `transition` プロパティで滑らかに

外部ライブラリ不要・Astroのバニラ環境のまま実装できるため、他のAstroプロジェクトにもそのまま流用可能です。
