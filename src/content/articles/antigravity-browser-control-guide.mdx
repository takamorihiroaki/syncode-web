## AIに「目」を授け、ブラウザを意のままに操る。

Antigravityの真骨頂である「Browser Control」が動かない、画面が真っ黒だ——そんな障壁で、AIの視覚を閉ざしてはいませんか？
動かない原因は、常に「接続・環境・表示」のいずれかに集約されます。

AIがサイトを縦横無尽に徘徊し、あなたの代わりに情報を集め、デバッグを行う。その圧倒的な利便性を解放するための最短手順を、ここに集約しました。

- <strong>MCP（Browser Control）を接続しているか</strong>
- <strong>Agent Manager（ワークスペース）を設定済みか</strong>
- <strong>ローカル環境（npm run dev）を起動しているか</strong>

👉 <strong>この3つが揃って初めて、AIはあなたのサイトを認識し、自動操作を開始します。</strong>

## Browser Controlが動かない本当の理由

結論：
👉 <strong>「接続」と「実行環境」がリンクしていないだけ</strong>

Browser Controlは単体では動きません。「道（接続）」「土台（設定）」「対象（起動中のサイト）」の3つが揃って初めて、AIの“目”が機能します。

---

## 手順（完全再現ステップ）

### Step 1：MCPを「Connected」にする（接続）

AIに「ブラウザを触る許可」を与えます。これが最大の詰まりポイントです。

1. 右上「<strong>Additional Options</strong>」→「<strong>MCP Servers</strong>」をクリック
2. 「<strong>Browser Control</strong>」を選択
3. 「<strong>Connect</strong>」をクリック

👉 <strong>「Connected」と緑色で表示されれば成功です。</strong>  
なるほどポイント①： <strong>「スイッチをONにするだけでなく、“接続中”の状態を確認してください」</strong>

---

### Step 2：Agent Managerを確認する（土台）

AIが「どのプロジェクトのサイトを見ればいいか」を認識させる必要があります。

1. 「<strong>Additional Options</strong>」→「<strong>Agent Manager</strong>」をクリック
2. ワークスペースが設定されているか、プロジェクトが表示されているか確認

👉 <strong>プロジェクトが表示されていない場合は、ワークスペースを再設定しましょう。</strong>  
なるほどポイント②： <strong>「作業場が特定できないと、Browser Controlは迷子になります」</strong>

---

### Step 3：ローカル環境を「起動」する（表示対象）

AIが見るための「本番のサイト」があなたのパソコンで動いている必要があります。

1. ターミナルを開く
2. `npm run dev` を実行し、サイトを起動する
3. ブラウザで実際にページが表示されていることを確認

👉 <strong>「表示されていないものは、AIも操作できません。」</strong>  
なるほどポイント③： <strong>「ブラウザでプレビューが動いていて初めて、AIはその画面を読み取れます」</strong>

## ⚠ よくある失敗

- <strong>「Connected」ではなく「Disconnected」のまま</strong>
  → 原因：接続エラー。一度ボタンを押し直して再接続しましょう。
- <strong>サイト（npm run dev）を起動し忘れている</strong>
  → 原因：AIが見る対象が存在しません。
- <strong>Agent Managerで別のプロジェクトを選んでいる</strong>
  → 原因：場所間違い。今作業しているフォルダを正しく設定しましょう。

ここが本質です。
