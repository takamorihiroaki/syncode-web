## その「はみ出し」は、プロの審美眼で解決できます。

PCでのプレビューでは完璧に見えたデザインが、iPhone 13 miniなどの横幅が狭い実機で確認した瞬間、無残に崩れてしまう。Web制作において最も「心折れる」瞬間に、あなたは直面してはいませんか？

原因は単なるピクセル不足ではなく、メディアクエリの精度の欠如にあります。Astroブログ特有のレイアウト崩れを一瞬で解消し、あらゆる端末で「美」を維持するためのCSS実装プロセスを公開します。

Web制作をやっていると一度は経験する、あるあるなトラブルです。

今回は、当SynCodeブログで実際に発生した「iPhone画面でのヘッダーメニューはみ出し問題」を、AIエージェント（Antigravity）と一緒にどのように解決したのか、具体的なCSSコードとともに記録します。

---

## 発生したトラブル：iPhone 13 miniでメニューが崩れる

事の発端は、実機での表示確認でした。
手元のiPhone 13 mini（画面幅：375px）でサイトを開いたところ、固定ヘッダーのナビゲーションメニューの右端にある「TOOLS」が、無残にもはみ出してしまっていたのです。

※以下は実際のスクリーンショットです。

![スクロール時のヘッダーが崩れた状態](/assets/IMG_0899.PNG)

原因は単純でした。
画面幅に対して、左側の「ロゴ（SYNCODE）」と右側の「メニュー4つ（TOP, ABOUT, WORK, TOOLS）」の<strong>合計幅が広すぎた</strong>ためです。特にスクロール時の固定ヘッダーにおいては、両者が1行に並ぶ設計になっていたため、横幅が足りずに「TOOLS」が押し出されてしまいました。

---

## 解決策：スマホ（480px以下）専用のCSSを追加する

この場合の正しいアプローチは、「要素を消す」ことではなく、「スマホ画面用に文字サイズと余白を詰める」ことです。

Astroの `BaseLayout.astro` ファイル内にあるグローバルCSS（`<style is:global>`）に、以下のような<strong>メディアクエリ</strong>（特定の画面幅の時だけ適用されるCSS）を追加しました。

### 実際に追加したCSSコード

```css
/* スマホ（iPhone 13 mini等）でのナビゲーションはみ出し対策 */
@media (max-width: 480px) {
    /* 1. 通常時のロゴサイズを少し小さく */
    .site-header .logo {
        font-size: 1.4rem;
    }
    
    /* 2. 通常時のメニューの文字サイズ・余白・文字間隔を詰める */
    .main-nav a {
        font-size: 0.75rem;
        padding: 6px 8px; /* 左右の余白を縮小 */
        letter-spacing: 0.08em;
    }
    
    /* 3. スクロール時のヘッダー全体の左右余白を減らす（8% → 4%） */
    .site-header.is-scrolled .header-inner {
        padding: 0 4%;
    }
    
    /* 4. スクロール時のロゴサイズをさらに小さく */
    .site-header.is-scrolled .logo {
        font-size: 0.95rem;
    }
    
    /* 5. スクロール時のメニューを極限まで詰める */
    .site-header.is-scrolled .main-nav a {
        font-size: 0.7rem;
        padding: 6px 5px; /* ここが最重要ポイント */
        letter-spacing: 0.05em;
    }
}
```

### 修正のポイント

- <strong>`max-width: 480px`</strong>: iPhone系のスマホはだいたい画面幅が375px〜430px程度なので、480px以下の画面に対してのみこのCSSが効くようにしています。
- <strong>`padding`（余白）の削減</strong>: 今回一番効果があったのはここです。ボタン化されている各メニューの左右の余白（padding）を限界まで削ることで、全体の横幅を劇的に節約しました。
- <strong>`letter-spacing`（文字間隔）の調整</strong>: `0.12em` など広めに取っていた文字間隔を `0.05em` に縮めることで、見た目を損なわずに幅を稼ぎました。

## AIエージェントを活用した、賢いデバッグ術

実は今回の修正、自分で一からCSSを計算したわけではありません。

1. はみ出している画面のスクリーンショットを撮る
2. AIエージェント（Antigravity）に画像を渡し、「スマホ（iPhone13mini）でメニューのはみ出し崩れが起きています。修正できますか？」と指示を出す。

これだけで、AIが現在の `BaseLayout.astro` のスタイルを解析し、最適なメディアクエリを計算して自動的にコードを適用してくれました。

<strong>「目で見えるバグは、スクショで伝えるのが一番早い」</strong>

これは、AIにコーディングを任せる上で非常に重要なテクニックです。言葉であれこれ説明するよりも、画像をポンと渡してしまった方が、AIは状況を一瞬で理解してくれます。

## まとめ：実機確認とメディアクエリはセットで考えよう

どんなにモダンなフレームワーク（AstroやReactなど）を使っていても、最終的な見た目を整えるのはCSSの役割です。

特にスマホの実機は「思っていたより狭い」ことが多々あります。「PCで見たら問題ないからOK」ではなく、一度は自分のスマホで確認し、必要であれば `@media (max-width: 480px)` で微調整する癖をつけておくと、UIの完成度が一段と上がります。
