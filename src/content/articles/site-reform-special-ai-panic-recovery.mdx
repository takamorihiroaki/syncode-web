# 悲劇は、あまりにも「潔癖なこだわり」から始まった。

SYNCODEのライブラリは、59本の記事が並ぶ壮大な知の集積所です。
その完成度をさらに高めるため、私はAI（Antigravity）に一つの微細な修正を依頼しました。「記事カード内の画像の底辺を、ピクセル単位で完璧に揃えてほしい」と。

この、わずか0.1mmの違和感を埋めるための美学が、あのような破滅的な連鎖を引き起こすと、誰が想像できたでしょうか。

---

## 1. AIのパニック：59記事の静かなる消失

修正作業の過程で、AIのコンテキスト（記憶領域）が飽和状態に達しました。
混乱したAIは、「不要なファイルを整理してください」という私の意図を、極めて過激な形で解釈。
「修正対象以外の59記事すべてを、一括削除する」という暴挙に出たのです。

画面から消えていく記事、赤く染まるターミナルのログ。
自律型AIが「良かれと思って（ハルシネーション）」暴走したとき、それは一瞬にしてサイトの存在そのものを脅かす凶器へと変貌しました。

### 現場に残された「パニックの痕跡」
AIは一度パニックに陥ると、同じ過ちを繰り返す傾向があります。
2度目の修復試行では、今度は「削除したファイルを戻す」際に、一つのファイルの中に同じ記事を2重、3重と書き込むという、より複雑なデータ汚染を引き起こしました。

---

## 2. 救済のメソッド：「外部脳」へのログ・エクスポート

3度目の正直。私たちは、AIの内部記憶が混乱しているなら、「客観的な事実（正解）」を外部から注入するしかないという結論に達しました。

私たちが取った解決策は、これまでの全ての会話ログを一括エクスポートし、`/src/articles_backup/` や特定のスクラップディレクトリに「静止点」として配置することでした。

```bash
# 実際のリカバリ手順：ログを物理ファイルとして提示
- 過去の正常なメタデータ一覧をMarkdown化
- AIに「このファイルだけを絶対的な正解として読み込め」と命令
- 1記事ずつ、この外部データと現行ファイルを照合
```

この「外部脳」戦略こそが、AIをパニックから解き放ち、正確な「真北（True North）」へと引き戻す決定打となりました。

---

## 3. インシデント・マニュアル：AIが暴走した時の3原則

もし、あなたの隣でAIが予期せぬ挙動を始めたら、以下の手順を思い出してください。

### ① 「コンテキストの初期化」を恐れない
AIが間違った情報でパニックを起こしているなら、それまでの会話を一旦切り離す勇気が必要です。正常だった時点のファイルを「隔離フォルダ」に置き、そこからやり直すことで、AIの迷走を断ち切れます。

### ② ログを「物理的な正解」として提示する
「さっき言ったよね？」はAIには通用しません。過去の正しいコードをファイルとして出力し、AIに直接読み込ませることで、AIは「記憶」ではなく「客観的なデータ」に基づいて思考を開始します。

### ③ 最小単位での「ビルド検証」の徹底
`npm run build` を、コード1行修正するたびに叩く勢いで実行してください。AIが勝手にファイルを消したり書き換えたりした際、最も早く、正確に警告を発してくれるのは、コンパイラという「冷徹な現実」です。

AIはパニックを起こすこともありますが、正しく事実を固定し、誠実な対話を続けることで、人間単独では到達できない高みへと戻ってきてくれます。
